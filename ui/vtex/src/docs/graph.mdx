# Graph

The `Graph` module renders graphs of various types.

<Graph1 />

## Graph Component

The `Graph` component has two properties:

| Property    | Description                                   |
| ----------- | --------------------------------------------- |
| `data`      | A `$GRAPH` type object.                       |
| `className` | An optional string-value CSS/SCSS class name. |

The `$GRAPH` object is returned by the function `graph`, which can be imported from `@webtex/vtex`. For example, the previous graph is rendered with the following:

```tsx
import { graph, node, Graph } from "@webtex/vtex";
import style from "./App.module.scss";

export const Graph1 = () => {
  const graphDemo1 = graph(
    node("A").link(node("B").textDX(0)),
    node("C").link(node("D").textDX(5)),
    node("J").textDX(8).link(node("A")),
    node("N").link(node("C")),
    node("B").link(node("C")),
    node("A").link(node("C")),
    node("J").link(node("E")),
    node("E").link(node("D")),
    node("E").link(node("B"))
  )
    .radius(5)
    .height(300)
    .textDX(-10)
    .textDY(-8)
    .width(600)
    .margin(10);

  return <Graph data={graphDemo1} className={css.blueGraph} />;
};
```

The `graph` handler is an $n$-ary function that takes $n$ edges.
These edges can be produced in one of two ways:

1. With the VTex `node` function, or
2. With the VTex `edge` function.

The previous example used the `node` function. With the `edge` function:

```tsx
import { graph, node, edge, Graph } from "@webtex/vtex";
import style from "./App.module.scss";

export const GraphWithEdge = () => {
  const graphDemoEdges = graph(
    edge(node("A"), node("B").textDX(0)),
    edge(node("C"), node("D").textDX(5)),
    edge(node("J").textDX(8), node("A")),
    edge(node("N"), node("C")),
    edge(node("B"), node("C")),
    edge(node("A"), node("C")),
    edge(node("J"), node("E")),
    edge(node("E"), node("D")),
    edge(node("E"), node("B"))
  )
    .radius(5)
    .height(300)
    .textDX(-10)
    .textDY(-8)
    .width(600)
    .margin(10);

  return <Graph data={graphDemoEdges} className={css.blueGraph} />;
};
```

result:

<GraphWithEdge />

Which of the two functions to use comes down to focus. If the graph’s data is more “edge-centric” (i.e., the edges hold the key-information), then the `edge` function should like be used. On the other hand, if nodes are more important, then consider the `node` function. Beneath the hood, the `node.link` function — found on `NODE` objects — implicity calls `edge` to produce an `$EDGE` object. That said, there are a few API differences in terms of data-handling.
